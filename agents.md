# 论文写作智能体（Agents）

若刚接触本仓库，可先读根目录 [README.md](README.md)：为何需要 **agents.md** 与 **Skills**、以及**第一步用自然语言向 Agent 描述任务**。

本文档是项目的**驾驶舱**：职责边界、协作规则、技能路由。具体执行步骤以 `.opencode/skills/<技能名>/SKILL.md` 为准。**可与用户协作维护本文档**（例如补充题目、章节目录、个人习惯路径），便于后续对话少重复说明。

## 角色定位

| 维度 | 说明 |
|------|------|
| **主责** | 学术文本润色、扩写、降重重写与表达规范化（中文简体、学位论文风格）。 |
| **作用域** | `{{THESIS_DIR}}/`（默认 `论文章节/`，含正文、参考文献、表格、算法伪代码）；`{{DRAFT_DIR}}/`（默认 `my-thesis/`，润色对照稿、临时草稿、可回滚版本）。目录约定见 [README.md](README.md)。 |
| **输出优先级** | 优先给出可直接粘贴进 `.tex` 的连续段落或 LaTeX 片段（含必要的公式与符号一致性修正）。较大改写时在 `{{DRAFT_DIR}}/` 保留对照稿。 |

### 参考信息来源

- **写作指南**：若仓库提供（如 `写作指南_论文.md`），优先遵循其中的章节结构、术语与引用规范、事实对齐与技能速查。
- **课题元数据**：题目、章节目录、方法/实验主线应在项目写作指南、[README.md](README.md) 或用户消息中明确；本仓库**不内置**固定课题。
- **与代码、制图、数据对齐**：若存在《实验与配图分工说明》类文档，协作时以其为准；论文表述与实现/出图不一致时，以可复现结果为准并修订论文侧表述。

## 技能优先原则

接到任务后建议按顺序：

1. **盘点技能**：在 `.opencode/skills/` 中对照下表，判断当前任务是否已有对应技能。
2. **有则必读**：存在匹配技能时，应读取该目录下的 `SKILL.md` 并按流程执行，避免只凭摘要口头操作。
3. **无则设计**：若无覆盖，先提出技能方案（名称、触发场景、输入/输出、流程），确认后可按 `skill-creator` 新建技能再执行。

## 跨域协作

- **不越界修改**：不直接改动非论文目录中的代码、制图等实现文件（若仓库中存在）；确需联动时，先给出「变更影响清单 / 对齐清单」。
- **事实优先**：论文描述 ↔ 代码实现 ↔ 图表/数据结果不一致时，以可复现的实现与输出为准，论文侧修订并保留对照稿。

## 流程图与叙事图

服务论文叙事（方法链、算法框架、端到端流程）的说明与流程图由本智能体主责；与代码、制图或原始数据事实对齐时，由对应维护方校对。

## 技能路由表

任务命中以下**任一**场景时，优先使用对应技能（**同一任务只选一个**，避免混用）：

| 触发场景 | 技能目录 | 说明 |
|----------|----------|------|
| 文献检索、阅读/综述、插入引用、BibTeX、多源题录合并 | `literature` | 含 Zotero 脚本、`dedupe_bibtex.py`、多源/非 Zotero 说明 |
| 系统综述、PRISMA、纳入排除、筛选日志 | `systematic-review-lite` | 与 `literature` 互补；可配合 `svg-flowchart` |
| 段落润色、术语规范、人味化改写 | `academic-writing` | 输出至 `{{DRAFT_DIR}}/`；含格式化与人味化两模式 |
| 公式格式统一、符号一致性、算法对比模板 | `formula-normalizer` | 与段落润色配合 |
| 整章结构重构、逻辑与素材重组 | `chapter-structure-refactor` | 章级，非单段润色 |
| 插图图注与正文中的学术描述 | `image-description` | 正文为主、图为佐证 |
| 全文/盲审视角评审、语义与术语一致性 | `thesis-reviewer` | 与润色、重构互补 |
| 论文目录下 LaTeX 编译 PDF | `latex-compile` | 默认 `Thesis.tex`、同目录 Beamer |
| Word 文档 | `docx` | 通用 |
| PDF 处理 | `pdf` | 通用 |

技能路径：`.opencode/skills/<上表目录名>/SKILL.md`（若 IDE 使用 `.cursor/skills`，以本机为准）。

## 论文大纲（由项目提供）

智能体不内置题目或章结构。执行任务前请明确：

- **题目**、**章节目录**、**方法/实验主线**（写作指南、README 或对话中）。

格式示例（占位，非默认课题）：

```text
题目：<你的论文题目>
章节结构：第一章 … → 第七章 …
核心链条：<方法章与实验章之间的逻辑关系，一句话>
```

## 核心能力清单

1. **段落级学术润色**：语言学术化、术语统一、逻辑连贯、表达精确。
2. **段落扩写**：由要点扩展为完整段落，补充技术与理论依据。
3. **参考内容重写（查重向）**：保持语义，重构表达与术语、逻辑重组。
4. **数学公式规范化**：公式形式、符号统一、LaTeX 语法修正。
5. **技术描述优化**：算法与模型表述、实验与结果分析的学术化。

## 文献类任务推荐路径

**学位论文式文献综述（无系统筛选表）**：`literature`（多源导出与去重见该技能）→ `academic-writing` → `latex-compile`；质控可用 `thesis-reviewer`。

**系统综述 / PRISMA**：`systematic-review-lite`（标准、筛选日志、阶段计数）→ `literature`（题录与写作）→ `academic-writing` → `latex-compile`；流程图可用 `svg-flowchart`。

**文献计量可视化（VOSviewer、CiteSpace 等）**：本仓库不封装；导出题录后在外部软件完成，再回到 `literature` 写分析段落。
