# Symbol Consistency

本文档提供符号一致性检查的详细规则和方法，用于确保全文数学符号使用的统一性和规范性。

## 符号一致性的重要性

### 为什么需要符号一致性

1. **提高可读性**：统一的符号使用让读者更容易理解内容
2. **避免混淆**：不一致的符号可能导致读者误解
3. **专业性体现**：符号一致性是学术写作的基本要求
4. **减少错误**：统一的符号使用可以减少因符号混乱导致的错误

### 常见的不一致问题

- 同一变量使用不同表示（如 `x_i` vs `x_{i}`）
- 同一运算符使用不同格式（如 `\times` vs `*`）
- 同一希腊字母使用不同表示（如 `\alpha` vs `α`）
- 同一向量使用不同标记（如 `\mathbf{x}` vs `\vec{x}`）

## 变量命名一致性

### 基本原则

1. **统一命名格式**
   - 同一变量在全文中使用相同的命名
   - 避免同义变量混用（如 `x` 和 `t` 表示同一概念）
   - 保持命名的逻辑性和系统性

2. **统一上下标格式**
   - 单字符上下标：`x_i`, `x^2`
   - 多字符上下标：`x_{max}`, `x^{2n}`
   - 避免混用（如 `x_i` 和 `x_{i}`）

3. **统一特殊标记**
   - 向量：`\mathbf{x}` 或 `\vec{x}`
   - 矩阵：`\mathbf{A}` 或 `\mathbf{M}`
   - 集合：`\mathcal{S}` 或 `\mathbb{S}`

### 常见变量命名规范

#### 索引变量

**标准格式**:
- 整数索引：`i, j, k, l, m, n`
- 向量索引：`i, j, k`
- 矩阵索引：`i, j`
- 时间索引：`t, k, n`

**示例**:
```markdown
$$
x_i = \sum_{j=1}^{n} a_{ij} y_j
$$
```

#### 向量和矩阵

**标准格式**:
- 向量：`\mathbf{x}`, `\mathbf{y}`, `\mathbf{z}`
- 矩阵：`\mathbf{A}`, `\mathbf{B}`, `\mathbf{M}`
- 单位向量：`\mathbf{e}_i`, `\mathbf{e}_j`

**示例**:
```markdown
$$
\mathbf{y} = \mathbf{A}\mathbf{x}
$$
```

#### 标量和常数

**标准格式**:
- 标量：`x, y, z, a, b, c`
- 常数：`c, k, \alpha, \beta`
- 参数：`\theta, \lambda, \mu`

**示例**:
```markdown
$$
y = \alpha x + \beta
$$
```

#### 集合和空间

**标准格式**:
- 实数集：`\mathbb{R}`
- 整数集：`\mathbb{Z}`
- 自然数集：`\mathbb{N}`
- 向量空间：`\mathcal{V}`, `\mathcal{W}`
- 解空间：`\mathcal{S}`, `\mathcal{X}`

**示例**:
```markdown
$$
x \in \mathbb{R}^n
$$
```

## 运算符格式一致性

### 基本原则

1. **统一运算符格式**
   - 同一运算符在全文中使用相同的格式
   - 避免混用不同表示（如 `\times` 和 `*`）
   - 根据文档风格选择合适的运算符

2. **专业化风格**
   - 使用LaTeX命令表示运算符
   - 乘法：`\times`
   - 除法：`\frac{a}{b}`
   - 比较：`\leq`, `\geq`, `\neq`

3. **简洁化风格**
   - 使用简化符号表示运算符
   - 乘法：`*`
   - 除法：`/`
   - 比较：`<=`, `>=`, `!=`

### 常见运算符规范

#### 算术运算符

**专业化风格**:
- 加法：`+`
- 减法：`-`
- 乘法：`\times` 或 `\cdot`
- 除法：`\frac{a}{b}`

**简洁化风格**:
- 加法：`+`
- 减法：`-`
- 乘法：`*`
- 除法：`/`

**示例**:
```markdown
专业化：$$a \times b + c \times d$$
简洁化：$$a * b + c * d$$
```

#### 比较运算符

**专业化风格**:
- 等于：`=`
- 不等于：`\neq`
- 小于：`<`
- 大于：`>`
- 小于等于：`\leq`
- 大于等于：`\geq`
- 约等于：`\approx`

**简洁化风格**:
- 等于：`=`
- 不等于：`!=`
- 小于：`<`
- 大于：`>`
- 小于等于：`<=`
- 大于等于：`>=`
- 约等于：`~`

**示例**:
```markdown
专业化：$$x \leq a, y \neq b$$
简洁化：$$x <= a, y != b$$
```

#### 逻辑运算符

**标准格式**:
- 与：`\land` 或 `\wedge`
- 或：`\lor` 或 `\vee`
- 非：`\neg` 或 `\lnot`
- 蕴含：`\Rightarrow` 或 `\implies`
- 等价：`\Leftrightarrow` 或 `\iff`

**示例**:
```markdown
$$
x \land y \Rightarrow z
$$
```

- [ ] 乘法运算符统一（`\times` vs `*`）

- [ ] 比较运算符统一（`\leq` vs `<=`）
- [ ] 风格一致（专业化 vs 简洁化）

## 希腊字母一致性

### 基本原则

1. **统一使用LaTeX命令**
   - 所有希腊字母必须使用LaTeX命令
   - 不能直接输入希腊字母字符
   - 大小写敏感

2. **统一大小写格式**
   - 小写：`\alpha, \beta, \gamma, ...`
   - 大写：`\Alpha, \Beta, \Gamma, ...`
   - 避免混用大小写表示同一概念

3. **统一斜体/正体使用**
   - 变量：斜体（默认）
   - 常数：正体（`\mathrm`）
   - 算子：正体（`\mathrm`）

### 希腊字母使用规范

#### 变量 vs 常数

**变量**（斜体）:
```markdown
$$
\alpha x + \beta y = \gamma
$$
```

**常数**（正体）:
```markdown
$$
\mathrm{\pi} r^2
$$
```

#### 算子 vs 变量

**算子**（正体）:
```markdown
$$
\Delta f = \nabla^2 f
$$
```

**变量**（斜体）:
```markdown
$$
\alpha + \beta = \gamma
$$
```

## 函数命名一致性

### 基本原则

1. **统一函数表示**
   - 标准函数使用LaTeX命令（如 `\sin`, `\cos`）
   - 自定义函数使用正体（如 `\mathrm{f}`）
   - 避免混用不同表示

2. **统一函数参数**
   - 函数参数使用括号包裹
   - 参数之间用逗号分隔
   - 保持参数命名的一致性

3. **统一函数调用格式**
   - 函数名后紧跟参数
   - 函数名与参数之间无空格
   - 多个函数调用格式一致

### 常用标准函数

#### 三角函数

| LaTeX | 显示 | LaTeX | 显示 |
|-------|------|-------|------|
| `\sin` | $\sin$ | `\cos` | $\cos$ |
| `\tan` | $\tan$ | `\cot` | $\cot$ |
| `\sec` | $\sec$ | `\csc` | $\csc$ |

**示例**:
```markdown
$$
y = \sin(x) + \cos(x)
$$
```

#### 反三角函数

| LaTeX | 显示 | LaTeX | 显示 |
|-------|------|-------|------|
| `\arcsin` | $\arcsin$ | `\arccos` | $\arccos$ |
| `\arctan` | $\arctan$ | `\arccot` | $\arccot$ |

**示例**:
```markdown
$$
\theta = \arctan\left(\frac{y}{x}\right)
$$
```

#### 双曲函数

| LaTeX | 显示 | LaTeX | 显示 |
|-------|------|-------|------|
| `\sinh` | $\sinh$ | `\cosh` | $\cosh$ |
| `\tanh` | $\tanh$ | `\coth` | $\coth$ |

**示例**:
```markdown
$$
y = \tanh(x)
$$
```

#### 对数和指数函数

| LaTeX | 显示 | LaTeX | 显示 |
|-------|------|-------|------|
| `\ln` | $\ln$ | `\log` | $\log$ |
| `\exp` | $\exp$ | `e^x` | $e^x$ |

**示例**:
```markdown
$$
y = \ln(x) + \exp(x)
$$
```

### 自定义函数

#### 命名规范

- 使用正体表示函数名：`\mathrm{f}`, `\mathrm{g}`
- 函数名通常为单个字母：`f, g, h`
- 特殊函数可以使用多个字母：`\mathrm{cost}`, `\mathrm{loss}`

**示例**:
```markdown
$$
y = \mathrm{f}(x) + \mathrm{g}(x)
$$
```

#### 函数参数

- 参数使用括号包裹：`\mathrm{f}(x)`
- 多个参数用逗号分隔：`\mathrm{f}(x, y)`
- 参数命名保持一致

**示例**:
```markdown
$$
z = \mathrm{f}(x, y) = x^2 + y^2
$$
```

- [ ] 自定义函数使用正体（`\mathrm{f}`）
- [ ] 函数参数使用括号包裹
- [ ] 函数名与参数之间无空格
- [ ] 函数调用格式统一

## 符号一致性检查方法

### 自动检查

#### 正则表达式匹配

1. **变量命名检查**
   ```regex
   # 检查同一变量的不同表示
   x_i|x_{i}
   x_max|x_{max}
   ```

2. **运算符格式检查**
   ```regex
   # 检查乘法运算符
   \\times|\*
   
   # 检查除法运算符
   \\frac|/
   
   # 检查比较运算符
   \\leq|<=
   ```

3. **希腊字母检查**
   ```regex
   # 检查希腊字母字符
   [α-ωΑ-Ω]
   ```

#### 频率统计

1. **统计变量使用频率**
   - 记录每个变量的出现次数
   - 识别不常用的变量
   - 检查是否有拼写错误

2. **统计运算符使用频率**
   - 记录每个运算符的出现次数
   - 识别混用的运算符格式
   - 检查是否需要统一

3. **统计希腊字母使用频率**
   - 记录每个希腊字母的出现次数
   - 识别混用的希腊字母格式
   - 检查大小写是否一致

### 手动检查

#### 逐项检查清单

1. **通读全文**
   - 识别所有变量
   - 识别所有运算符
   - 识别所有希腊字母
   - 识别所有函数

2. **建立符号表**
   - 列出所有使用的符号
   - 记录每个符号的含义
   - 记录每个符号的格式

3. **对比检查**
   - 检查同一符号是否有不同格式
   - 检查是否有未定义的符号
   - 检查是否有重复定义的符号

#### 符号表模板

```markdown

|------|------|-------------|
```

## 符号统一建议

### 统一策略

1. **确定统一标准**
   - 根据文档风格选择符号格式
   - 专业化风格：使用完整LaTeX命令
   - 简洁化风格：使用简化符号

2. **建立符号规范**
   - 制定变量命名规范
   - 制定运算符格式规范
   - 制定希腊字母使用规范

3. **全局替换**
   - 使用编辑器的全局替换功能
   - 逐项替换符号格式
   - 替换后检查是否正确

### 替换示例

#### 变量命名统一

**替换前**:
```markdown
x_i + x_{i} + x_max
```

**替换后**:
```markdown
x_i + x_i + x_{max}
```

#### 运算符格式统一

**替换前**（专业化）:
```markdown
a * b + c / d
```

**替换后**（专业化）:
```markdown
a \times b + \frac{c}{d}
```

#### 希腊字母统一

**替换前**:
```markdown
α + β = γ
```

**替换后**:
```markdown
\alpha + \beta = \gamma
```
