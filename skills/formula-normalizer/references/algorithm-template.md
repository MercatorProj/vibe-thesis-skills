# Algorithm Template

本文档提供算法对比描述的完整模板和示例，用于生成学术级的算法对比内容。

## 适用场景

在硕士/博士论文的以下章节中，需对两种或多种基础算法/数值方法/求解策略进行正式、结构化、可比较的介绍：

- **引言** - 介绍研究背景和相关方法
- **研究背景** - 阐述领域内常用方法
- **相关工作** - 回顾和对比现有方法
- **方法论** - 详细描述所采用的方法

**应用领域**：
- 计算力学、控制工程、金融数学
- 地球物理、生物建模、能源系统
- 机器学习底层优化器等任何涉及数值计算的学科

**核心特征**：问题导向 + 方法分述 + 公式支撑 + 对比总结

## 输入要素

为生成高质量的算法对比描述，需要用户提供以下结构化信息：

| 要素 | 说明 | 示例 |
|------|------|------|
| **研究问题** | 所要解决的具体数值/计算问题 | "求解稳态热传导偏微分方程的隐式离散系统" |
| **变量性质** | 强调是**连续值**（实数域）求解 | "状态变量 $u \in \mathbb{R}^n$ 为连续场量" |
| **待描述方法列表** | 明确需介绍的算法名称（2–3种为宜） | "共轭梯度法、雅可比迭代法" 或 "二分法、牛顿法" |
| **是否需公式** | 是否包含关键公式及符号说明 | "是，所有公式需编号并附符号注释" |
| **结构偏好** | 是否要求"原理→公式→步骤→对比"流程 | "按方法逐一展开，最后统一比较" |

## 输出规范

生成内容须满足以下标准：

### 语言风格
- 正式、客观、逻辑严密
- 符合中文学术论文规范
- 使用准确的学术术语

### 整体结构

1. **总起段**
   - 阐明研究问题背景
   - 说明连续性特征
   - 引入所述方法的必要性

2. **方法A描述**
   - 自然段阐述**基本原理与思想来源**
   - 给出**核心公式或迭代格式**（带编号）
   - **紧接公式下方**提供**符号说明表**
   - 自然段概述**关键实施步骤与终止条件**

3. **方法B描述**
   - 同方法A的结构
   - 可扩展至方法C

4. **对比分析段**
   - 从以下维度进行客观比较：
     - 收敛性（全局/局部、阶数）
     - 计算复杂度（每步成本、内存需求）
     - 鲁棒性（对初值、参数、噪声的敏感性）
     - 适用前提（如光滑性、可导性、单调性）
     - 在当前应用领域的适配性

### 数学表达
- 使用LaTeX格式
- 变量首次出现时应有语义定义
- 避免未解释的缩写（如首次出现"PDE"应写"偏微分方程（Partial Differential Equation, PDE）"）

### 领域融合
- 将方法动机与应用场景自然结合
- 避免"方法孤岛"
- 体现问题驱动的动机

## 示例1：二分法 vs 牛顿法

### 完整示例

```markdown
在数值计算领域，非线性方程求根问题通常归结为对连续变量 $x \in \mathbb{R}$ 的求解任务，其数学形式常表现为非线性代数方程 $f(x) = 0$。针对此类问题，二分法与牛顿迭代法作为两类具有代表性的数值策略，因其各自的优势，被广泛应用于科学计算和工程应用的相关研究中。

二分法是一种基于区间收缩的确定性方法，其核心思想是利用连续函数的介值定理，通过不断缩小包含根的区间来逼近真实解。该方法特别适用于函数不可导但连续的情形，对函数的光滑性要求较低。

二分法的关键更新规则如下：

$$
[a_{k+1}, b_{k+1}] = 
\begin{cases}
[a_k, c_k] & \text{if } f(a_k) \cdot f(c_k) < 0 \\
[c_k, b_k] & \text{if } f(c_k) \cdot f(b_k) < 0
\end{cases}
\tag{1}
$$

其中：
- $[a_k, b_k]$ 为第 $k$ 次迭代的区间，满足 $f(a_k) \cdot f(b_k) < 0$；
- $c_k = \frac{a_k + b_k}{2}$ 为区间中点；
- $f(\cdot)$ 为目标函数。

其实现通常包含以下步骤：（1）选取初始区间 $[a_0, b_0]$，确保函数在区间端点异号；（2）计算区间中点 $c_k$ 及其函数值 $f(c_k)$；（3）根据函数值符号更新区间；（4）判断收敛条件（如 $|b_k - a_k| < \varepsilon$ 或 $|f(c_k)| < \delta$），若满足则终止，否则继续迭代。该方法具有全局线性收敛性，其主要优势在于鲁棒性强、无需导数信息，但收敛速度较慢，每次迭代仅将区间长度减半。

相比之下，牛顿迭代法则是一种基于局部线性化思想的高阶数值方法，适用于函数光滑且导数可计算的情形。其核心在于利用函数在当前近似点处的一阶泰勒展开构造切线，并以切线与横轴的交点作为下一轮迭代的估计值。在车辆动力学中，当涉及如纵向加速度平衡、稳态转向角求解等可导非线性方程时，牛顿法能显著提升收敛效率。

牛顿迭代法的迭代公式为：

$$
x_{k+1} = x_k - \frac{f(x_k)}{f'(x_k)}
\tag{2}
$$

其中：
- $x_k$ 为第 $k$ 次迭代的近似解；
- $f(x_k)$ 为函数在 $x_k$ 处的取值；
- $f'(x_k)$ 为函数在 $x_k$ 处的一阶导数，且要求 $f'(x_k) \neq 0$。

该方法的关键步骤包括：（1）选取初始猜测值 $x_0$，通常基于物理先验知识（如车辆稳态运行工况下的合理参数范围）；（2）在每一步迭代中计算函数值 $f(x_k)$ 及其导数 $f'(x_k)$；（3）利用公式 (2) 更新近似解；（4）判断收敛条件（如 $|x_{k+1} - x_k| < \varepsilon$ 或 $|f(x_{k+1})| < \delta$），若满足则终止，否则继续迭代。在函数 $f$ 二阶连续可导且初值 $x_0$ 充分接近真实根的条件下，牛顿法具有局部二次收敛性，即误差平方级衰减，远快于二分法。

综合而言，二分法与牛顿迭代法在解决非线性方程求根问题时呈现出明显的互补性。二分法以全局收敛和鲁棒性见长，适合初值未知或函数性质复杂的场景；而牛顿法则在初值质量良好且函数光滑时展现出超线性收敛优势，但需承担对初值敏感和导数计算成本的风险。因此，在实际工程或科研实践中，常采用混合策略（如外层用二分法保证收敛，内层用牛顿法加速），以在稳定性与效率之间取得平衡。
```

## 示例2：梯度下降 vs 共轭梯度

### 完整示例

```markdown
在优化计算领域，无约束优化问题通常归结为对连续变量 $\mathbf{x} \in \mathbb{R}^n$ 的求解任务，其数学形式常表现为最小化目标函数 $\min f(\mathbf{x})$。针对此类问题，梯度下降法与共轭梯度法作为两类具有代表性的数值策略，因其各自的优势，被广泛应用于机器学习、数值分析和工程优化的相关研究中。

梯度下降法是一种基于一阶梯度信息的最优化方法，其核心思想是沿着目标函数在当前点的负梯度方向进行迭代更新，以逐步逼近局部最小值。该方法特别适用于大规模优化问题，对计算资源要求较低。

梯度下降法的关键更新规则如下：

$$
\mathbf{x}_{k+1} = \mathbf{x}_k - \alpha_k \nabla f(\mathbf{x}_k)
\tag{1}
$$

其中：
- $\mathbf{x}_k$ 为第 $k$ 次迭代的参数向量；
- $\alpha_k$ 为第 $k$ 次迭代的学习率（步长）；
- $\nabla f(\mathbf{x}_k)$ 为目标函数在 $\mathbf{x}_k$ 处的梯度向量。

其实现通常包含以下步骤：（1）初始化参数向量 $\mathbf{x}_0$ 和学习率 $\alpha$；（2）在每一步迭代中计算目标函数的梯度 $\nabla f(\mathbf{x}_k)$；（3）利用公式 (1) 更新参数向量；（4）判断收敛条件（如 $\|\nabla f(\mathbf{x}_{k+1})\| < \varepsilon$ 或 $\|\mathbf{x}_{k+1} - \mathbf{x}_k\| < \delta$），若满足则终止，否则继续迭代。该方法具有线性收敛性，其主要优势在于实现简单、每步计算成本低，但容易陷入局部最优且收敛速度受学习率影响较大。

相比之下，共轭梯度法则是一种利用共轭方向信息的高阶优化方法，适用于目标函数为二次凸函数或近似二次凸函数的情形。其核心思想是通过构造相互共轭的搜索方向，避免梯度下降法中的"之字形"震荡，从而加速收敛。在训练深度神经网络时，当目标函数近似为二次凸函数时，共轭梯度法能显著提升优化效率。

共轭梯度法的迭代格式可表示为：

$$
\mathbf{x}_{k+1} = \mathbf{x}_k + \alpha_k \mathbf{d}_k
\tag{2}
$$

其中：
- $\mathbf{d}_k$ 为第 $k$ 次迭代的共轭搜索方向，定义为 $\mathbf{d}_k = -\nabla f(\mathbf{x}_k) + \beta_k \mathbf{d}_{k-1}$；
- $\beta_k$ 为共轭系数，常用计算公式为 $\beta_k = \frac{\nabla f(\mathbf{x}_k)^T \nabla f(\mathbf{x}_k)}{\nabla f(\mathbf{x}_{k-1})^T \nabla f(\mathbf{x}_{k-1})}$（Fletcher-Reeves公式）；
- $\alpha_k$ 为沿方向 $\mathbf{d}_k$ 的最优步长，通常通过线搜索确定。

典型实施流程包括：（1）初始化参数向量 $\mathbf{x}_0$ 和初始搜索方向 $\mathbf{d}_0 = -\nabla f(\mathbf{x}_0)$；（2）在每一步迭代中计算梯度 $\nabla f(\mathbf{x}_k)$ 和共轭系数 $\beta_k$；（3）更新搜索方向 $\mathbf{d}_k$；（4）通过线搜索确定最优步长 $\alpha_k$；（5）利用公式 (2) 更新参数向量；（6）判断收敛条件。在目标函数为严格凸二次函数的条件下，该方法具有有限步收敛性（最多 $n$ 步收敛），但对目标函数的凸性和光滑性要求较高。

综合而言，梯度下降法与共轭梯度法在解决无约束优化问题时呈现出明显的互补性。梯度下降法以实现简单和鲁棒性见长，适合大规模优化和目标函数复杂的场景；而共轭梯度法则在目标函数近似为二次凸函数时展现出超线性收敛优势，但需承担对函数性质要求较高和实现复杂度增加的风险。因此，在实际工程或科研实践中，常采用混合策略（如前期用梯度下降快速下降，后期用共轭梯度精细收敛），以在稳定性和效率之间取得平衡。
```

## 注意事项与最佳实践

### 术语准确性

- **区分相似概念**：
  - "二分查找"用于离散数组 vs "二分法"用于连续求根
  - "梯度下降"用于优化 vs "牛顿法"用于方程求根
  - "显式方法" vs "隐式方法"在数值求解中的区别

- **准确使用数学术语**：
  - "收敛性" vs "稳定性"
  - "局部收敛" vs "全局收敛"
  - "线性收敛" vs "超线性收敛" vs "二次收敛"

### 收敛性表述严谨

- **明确收敛条件**：
  - "全局收敛"需额外条件（如凸性、Lipschitz连续）
  - "局部收敛"需初值充分接近真实解
  - "二次收敛"需函数二阶连续可导

- **避免过度承诺**：
  - 不说"必定收敛"，而说"在满足...条件下收敛"
  - 不说"最快"，而说"在...条件下具有优势"
  - 不说"总是有效"，而说"在...场景中适用"

### 领域贴合技巧

- **加入领域语境**：
  - "在电池管理系统中，由于开路电压-荷电状态映射常呈非光滑特性，二分法成为可靠选择"
  - "在车辆动力学中，当涉及如纵向加速度平衡、稳态转向角求解等可导非线性方程时，牛顿法能显著提升收敛效率"
  - "在训练深度神经网络时，当目标函数近似为二次凸函数时，共轭梯度法能显著提升优化效率"

- **体现问题驱动**：
  - 说明方法是为了解决什么具体问题而设计的
  - 说明方法在特定应用场景中的优势
  - 避免孤立地描述方法

### 可扩展性

- **扩展至三方法对比**：
  - 复制"方法描述"模块
  - 调整对比分析段，加入第三方法的比较
  - 在综合分析中讨论三方法的互补性

- **扩展至更多方法**：
  - 可以扩展到4-5种方法的对比
  - 建议按方法类别分组（如一阶方法、二阶方法、启发式方法）
  - 在对比分析中使用表格总结

### 避免绝对优劣

- **强调适用条件决定性能**：
  - 不说"方法A优于方法B"
  - 而说"方法A在...条件下具有优势，方法B在...条件下具有优势"
  - 强调"互补性"而非"优劣性"

- **客观比较**：
  - 从多个维度（收敛性、复杂度、鲁棒性）客观比较
  - 避免主观评价
  - 提供具体的数据或理论支持

### 符号说明规范

- **符号表格式**：
  - 使用列表形式（如"其中："后跟列表）
  - 每个符号占一行
  - 符号和说明之间用冒号或空格分隔

- **符号定义时机**：
  - 变量首次出现时应定义
  - 公式下方应提供完整的符号说明表
  - 避免读者需要回溯查找符号定义

### 公式编号规范

- **编号格式**：
  - 使用 `\tag{编号}` 命令
  - 编号应连续（如1, 2, 3...）
  - 在正文中引用时使用"(编号)"格式

- **编号位置**：
  - 放在公式末尾
  - 与公式之间用空格分隔
  - 不影响公式显示

- [ ] 结构清晰，逻辑连贯

- [ ] 公式正确，符号说明完整

- [ ] 对比分析客观，维度全面

- [ ] 无绝对优劣表述，强调适用条件
