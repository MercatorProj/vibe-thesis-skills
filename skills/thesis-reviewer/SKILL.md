# 学位论文评审与语义审查

**学术诚信与「人味化/风格审查」的正当用途**见仓库根目录 [DISTRIBUTION.md](../../../DISTRIBUTION.md)。Part B 跨章主题清单见 [references/review-checklist.md](references/review-checklist.md)（支持自建条目 + 可替换示例）。

## 审查模式

根据用户需求选择执行模式：

| 模式 | 触发场景 | 执行内容 |
|------|---------|---------|
| **全面评审** | "盲审""评审""全面检查" | Part A（宏观评审）+ Part B（语义精审）+ **Part C（AI 痕迹审查，默认执行）** |
| **宏观评审** | "评审意见""论文质量""能否毕业" | 仅执行 Part A |
| **语义精审** | "语义重复""术语一致""冗余检查""跨章对比" | 仅执行 Part B |
| **AI风格审查** | "AI风格""AI痕迹""人味化检查""写作风格审查""去AI味" | 仅执行 Part C（也可在全面评审/单章审查中默认包含） |
| **单章审查** | "审查第N章""检查这一节" | 指定章的 Part B 第一轮 + **Part C（AI 痕迹审查，默认执行）**，以及 Part A 维度4-5 |

## Part A：宏观评审（导师/盲审视角）

### 评审维度

#### A1 选题与综述
- **选题价值**：是否具有明确的理论意义或工程应用价值，是否紧扣学科前沿或国家/行业需求
- **综述质量**：文献调研是否全面、客观；引用是否权威、时效性强
- **问题导向**：是否基于综述准确指出不足，顺理成章引出科学问题

#### A2 理论基础与研究方法
- **概念准确性**：基本概念界定是否清晰，理论基础是否扎实
- **方法科学性**：模型、算法、实验方法是否科学合理
- **技术路线**：总体技术路线是否可行，逻辑链条是否严密

#### A3 工作量与研究成果
- **工作饱满度**：实验数据量、系统开发规模、案例分析深度是否达标
- **论证与分析**：是否深入到机理层面，而非仅描述现象
- **创新与贡献**：是否有理论/方法/应用层面的创新，是否有说服力

#### A4 论文逻辑与结构
- **篇章结构**：章节安排是否合理（绪论→理论→建模→验证→总结），章间逻辑关联是否强
- **段落连贯**：段内及段间过渡是否自然，有无逻辑跳跃、论证断层或前后矛盾

#### A5 学术写作与格式规范
- **语言表达**：是否严谨、简练、客观，有无口语化或主观臆断
- **图表质量**：清晰度、标签、单位、图注表注是否具备自明性
- **排版规范**：公式编号、引用格式、术语前后一致性

### 宏观评审产出

1. **总体评价**：最大亮点 + 最致命缺陷 + 是否达到学位水平
2. **分维度修改意见**：指出问题并说明原因
3. **落地建议**：具体指导如何补充数据/对比实验/重绘图表/重构逻辑

## Part B：语义精审（逐章 + 跨章 + 术语）

### 问题分类标记

| 标记 | 类别 | 定义 |
|------|------|------|
| REP | 语义重复 | 同一事实/结论在不同位置被重复陈述 |
| RED | 语义冗余 | 段内信息量低、同义反复、废话修饰 |
| CON | 表述冲突 | 同一概念在不同位置用了不兼容的说法 |
| TRM | 术语不一致 | 同一变量/缩写/术语名称不统一 |
| LNK | 衔接断裂 | 章首/章尾过渡与相邻章节内容脱节 |
| FMT | 格式瑕疵 | 公式环境、引用格式、图表编号等排版问题 |

### 三轮审查流程

#### 第一轮：章内审查

逐章读取全部 .tex 文件，对每章产出问题清单。

**执行顺序**：核心技术章优先（第3→4→5→6→2→1→7章）。

**检查项**：
1. 逐段扫描段内冗余（同义反复、无信息量修饰）
2. 节间重复（同一章不同节重复描述相同内容）
3. 章内术语与符号一致性
4. 各节首尾过渡衔接
5. 论证深度（是否仅描述现象而缺乏机理分析）

#### 第二轮：跨章审查

针对高频重复/冲突域，同时读取两章相关节做对比。

**通用跨章审查主题**：
- 核心概念定义是否在多章重复从零解释
- 算法/方法描述是否在综述、正文、案例中各写一遍
- 问题定义（数学模型）是否在建模章和案例章重复
- 流程描述是否在多处被重复叙述
- 研究区/实验背景是否在数据章和案例章重复
- 参数值/名称在建模章、仿真章、附录中是否一致
- "基线方法"等对比概念的定义是否全文一致

项目专属的跨章审查主题见 [review-checklist.md](references/review-checklist.md)。

#### 第三轮：术语与符号一致性

建立术语-符号对照表，全文核对：
1. 核心术语中文表述统一性
2. 数学符号写法统一性（如 `$v_{\max}$` vs `$v_{\text{max}}$`）
3. 英文缩写首次全称 + 后续统一缩写
4. 图表编号引用有效性（`\ref` 是否指向已删除的图表）
5. 量纲与单位全文统一（m/s vs km/h 等）

### 辅助操作：关键词快速扫描

精审前可先用关键词搜索定位高频重复区域：
- 对每个可疑术语变体在全文 .tex 中搜索出现次数和位置
- 标记出现在 3 个以上文件中的术语为"跨章重复候选"

## Part C：AI风格审查

与 academic-writing 的 **humanizer-zh** 规则一致，规则详见 [../academic-writing/references/humanizer-zh-patterns.md](../academic-writing/references/humanizer-zh-patterns.md)。执行 Part C 时读取该 reference，使用其中的模式清单与检查项。**全面评审**与**单章审查**时**默认执行** Part C，无需用户单独提出「AI 风格」；用户也可仅要求「仅 AI 风格审查」单独跑 Part C。旨在识别典型 AI 辅助写作的文体特征，便于针对性改写、提升自然度与学术人味，**不用于判定是否抄袭**，仅作写作质量与风格优化参考。

### 检查维度

| 维度 | 说明 | 典型表现 |
|------|------|----------|
| **机械过渡** | 过渡词/连接词过于集中、可预测 | 「首先、其次、此外、然而、综上所述」等成串出现；段首频繁「鉴于此」「在此基础上」「从…来看」 |
| **句长均匀** | 句子长度缺乏节奏变化 | 多数句子长度接近，缺少短句点睛或长句铺陈 |
| **抽象套话** | 信息量低的万能句式 | 「具有重要意义」「值得关注」「在…方面」「从某种意义上说」「值得注意的是」等空泛表述 |
| **AI惯用词** | 常见于 AI 生成文本的流行词、套话词 | 如「闭环」「赋能」「抓手」「落地」「痛点」「打通」「迭代」「沉淀」「维度」「场景」「链路」「底层逻辑」「端到端」「深度融合」「有机结合」「有效/显著提升」「系统性」「整体性」「协同」「耦合」「范式」「可复现」「可扩展」等；若与学科习惯不符或显得空泛则标出 |
| **惯用句式与结构** | 固定搭配的句式和「A-B-C」式并列结构 | 「首先…其次…最后…」「一是…二是…三是…」；「在…基础上/背景下/框架下」连用；「从…到…的完整链路/闭环」；「实现了…与…的有机结合/深度融合」；「为…提供了…/奠定了…」；每段以「本文」「本研究」开头；过度使用「可以」「能够」「能够有效」等 |
| **词汇单一** | 同一概念反复用同一词、缺乏同义替换 | 同一段内多次重复同一动词/名词；术语外词汇多样性低 |
| **并列结构过工整** | 刻意对称的句式 | 「一方面…另一方面」成对出现；连续多句「…，从而…」「…，进而…」 |
| **段落结构雷同** | 段内结构过于规整 | 每段均为「总—分—总」或「现象—原因—结论」的固定套路 |

**AI 惯用词与惯用句式示例（审查时可对照）：**

- **惯用词**：闭环、赋能、抓手、落地、痛点、打通、迭代、沉淀、维度、场景、链路、底层逻辑、端到端、深度融合、有机结合、有效提升、显著提升、系统性、整体性、协同、耦合、范式、可复现、可扩展、全方位、一站式、颗粒度（非专业语境）。
- **惯用句式/结构**：首先—其次—最后 / 一是—二是—三是；在…的基础上、在…的背景下、在…的框架下；从…到…的完整…；实现了…与…的…；为…提供了…、为…奠定了基础；段首反复「本文…」「本研究…」；连续「可以…」「能够…」「能够有效…」。

### 问题标记

| 标记 | 类别 | 定义 |
|------|------|------|
| AIT | AI典型过渡 | 机械过渡词集中、可替换为更自然的衔接 |
| AIS | AI典型句式 | 抽象套话、万能句式，建议改为具体表述 |
| AIV | AI惯用词 | 出现常见 AI 流行词/套话词（如闭环、抓手、落地等），建议按学科习惯替换或删除 |
| AIF | 惯用句式/结构 | A-B-C 式并列、固定搭配句式（如「在…基础上」连用、「为…奠定…」），建议拆解或改写 |
| AIW | 词汇单一 | 同段内重复用词过多，建议同义替换或省略 |
| AIP | 结构雷同 | 段落或句群结构过于工整、套路化 |

### 执行方式

1. **逐章扫描**：按 3→4→5→6→2→1→7 章顺序，对每章正文（不含图表、公式）做段落级扫描。
2. **对照 humanizer-zh**：结合 [../academic-writing/references/humanizer-zh-patterns.md](../academic-writing/references/humanizer-zh-patterns.md) 中的内容/语言/风格/交流模式与快速检查清单，识别命中项。
3. **标注位置**：对疑似处标注文件:行号（或节号）、类型（AIT/AIS/AIV/AIF/AIW/AIP，或 humanizer-zh 对应模式）、原文片段与简短修改建议。
4. **可选工具**：若项目内存在 `academic-writing` 技能的 `scripts/ai_detector.py`、`scripts/text_analyzer.py`，可对导出段落运行检测（如句长分布、过渡词密度），辅助定位高风险段落；结果与人工审查结合使用。

### 产出格式

AI风格审查产出可并入 Part B 问题清单表，**类型**列使用 AIT/AIS/AIV/AIF/AIW/AIP；**建议修改**列给出具体改写方向（如「改为具体结论/数据」「缩短句长」「替换过渡词」「替换惯用词」「拆解 A-B-C 结构」）。可增加**AI 痕迹诊断**小结：列出命中 humanizer-zh 模式的典型段落与对应建议。亦可单独输出「AI风格审查清单」，格式同问题清单表。

## 产出格式

### 问题清单表（Part B 每轮产出；Part C 可并入或单独成表）

```markdown
| 编号 | 类型 | 位置（文件:行号） | 问题描述 | 建议修改 | 优先级 |
|------|------|------------------|---------|---------|--------|
| R01  | REP  | 3.2.0:45 & 4.2.0:12 | 两处完整解释坡度阻力公式 | 第4章仅引用第3章 | 高 |
| AI01 | AIT  | 4.2.0:12 | 段首连续「首先」「其次」 | 改为自然衔接或合并句式 | 中 |
```

**类型**：Part B 使用 REP/RED/CON/TRM/LNK/FMT；Part C 使用 AIT/AIS/AIV/AIF/AIW/AIP。

**优先级**：高（影响学术严谨性/审稿人会直接指出）、中（影响阅读体验）、低（细节瑕疵）。

### 全面评审报告（Part A + B 汇总时产出）

1. 总体评价与评分（按 A1-A5 五维度）
2. 关键问题汇总（从 Part B 问题清单中提取高优先级项）
3. **AI 痕迹诊断**（Part C）：若已执行 Part C，汇总命中 humanizer-zh 模式的典型段落与修改建议
4. 修改优先级排序与落地建议

---

## 执行流程

1. **全局概览**：快速通读摘要、绪论和结论，把握研究目标与核心结论
2. **宏观评审**（Part A）：按 A1-A5 五维度交叉检查（全面评审时执行）
3. **语义精审**（Part B）：按三轮流程逐步执行（全面评审或语义精审时执行）
4. **AI风格审查**（Part C）：按 humanizer-zh 规则与检查维度逐章扫描；**全面评审与单章审查时默认执行**，也可应用户要求仅执行 Part C
5. **汇总输出**：合并所有问题清单，按优先级排序，输出完整审查报告（含 AI 痕迹诊断小结，若已执行 Part C）
