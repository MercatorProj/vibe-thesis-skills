# Chapter-Specific Strategies（通用）

本文档按**章节在论文中的功能**给出重构侧重点，**不绑定**具体学科或固定章号。使用时将你的章映射到下列类型，再结合 `refactor-types.md` 执行。

## 理论与背景类（常见于绪论、文献综述、问题提出）

- **目标**：压缩重复综述、突出问题与贡献边界、与后文方法/实验对齐。
- **重构要点**：按「领域共识 → 不足 → 本文切入点」重排；同一文献不在综述与后文重复展开。

## 数据与设定类（常见于数据章、实验环境）

- **目标**：符号、数据来源、预处理与后文模型输入一致。
- **重构要点**：与模型章、实验章交叉引用（定义一次、后文引用编号）；避免在案例章重复整段环境描述。

## 模型与方法类（常见于核心方法章）

- **目标**：形式化定义、假设、算法或模型结构清晰，且与符号表、图示一致。
- **重构要点**：按「问题形式化 → 方法主体 → 复杂度/对比」组织；删除与综述重复的「常识性算法介绍」，改为引用文献或附录。

## 实验与案例类（常见于验证、实例、应用）

- **目标**：可复现设置、公平对比、结果与主张一一对应。
- **重构要点**：基线/指标/数据集名称全文统一；图表解读不重复方法章推导；突出与本文贡献直接相关的对照。

## 结论与展望类

- **目标**：与摘要、贡献表述一致，不引入正文未支持的新主张。
- **重构要点**：贡献条目前后一致；局限与展望与第1章问题呼应。

## 跨章链条（自行替换为你的课题）

若论文存在「A → B → C」的依赖（例如环境建模 → 决策方法 → 实证），重构时检查：

1. 后序章是否仅引用前序章结论与编号，而非重述推导。
2. 同一图表/符号是否在首次出现处定义清楚。

将上述映射写入你的项目写作指南后，执行本技能时以该指南为准。
