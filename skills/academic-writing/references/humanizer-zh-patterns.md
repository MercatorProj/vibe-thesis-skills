# Humanizer-zh：去 AI 痕迹规则（论文润色与审查共用）

本文件与 thesis-reviewer 的 Part C（AI 风格审查）共用，作为论文正文润色、改写与审查时的**默认去 AI 痕迹 + 人味化**规则。来源：维基百科「AI 写作特征」综合指南。

---

## 核心规则（5 条）

1. **删除填充短语** — 去除开场白和强调性拐杖词
2. **打破公式结构** — 避免二元对比、戏剧性分段、修辞性设置
3. **变化节奏** — 混合句子长度；两项优于三项；段落结尾多样化
4. **信任读者** — 直接陈述事实，跳过软化、辩解和手把手引导
5. **删除金句** — 若听起来像可引用语句，重写它

---

## 内容模式与关键词

| 模式 | 需警惕的词汇/结构 |
|------|-------------------|
| **1. 夸大意义/遗产/趋势** | 作为/充当、标志着、见证了、是…的体现/证明/提醒、极其重要/至关重要/核心/关键性的作用/时刻、凸显/强调/彰显、反映更广泛的、象征…持续/永恒、为…奠定基础、关键转折点、不断演变的格局、深深植根于 |
| **2. 过度强调知名度** | 独立报道、由知名专家撰写、活跃的社交媒体 |
| **3. 肤浅分析（-ing 式）** | 突出/强调/彰显…、确保…、反映/象征…、为…做出贡献、培养/促进…、涵盖…、展示… |
| **4. 宣传/广告语** | 拥有（夸张）、充满活力的、丰富的（比喻）、深刻的、增强其、展示、体现、致力于、自然之美、坐落于、开创性的、令人叹为观止、必游之地、迷人的 |
| **5. 模糊归因** | 行业报告显示、观察者指出、专家认为、一些批评者认为、多个来源（却少具体引用） |
| **6. 公式化「挑战与展望」** | 尽管…面临若干挑战、尽管存在这些挑战、挑战与遗产、未来展望 |

**6.5 结构公式化（节/段层面）**

| 模式 | 需警惕的表现 |
|------|----------------|
| **每节同构** | 每节段落数相同、每节都是「总—分—总」或「现象—原因—结论」 |
| **列表项数雷同** | 连续多节均为三项或四项列表，无变化 |
| **重复收束句式** | 每段都以「综上所述」「值得注意的是」「鉴于此」等同类句式结尾；或英文 "What this means", "The takeaway" 式 callout |
| **每段都打蝴蝶结** | 每段结尾都是总结句，缺少自然收束或留白 |

## 语言与语法模式

| 模式 | 需警惕的词汇/结构 |
|------|-------------------|
| **7. AI 高频词** | 此外、与…保持一致、至关重要、深入探讨、强调、持久的、增强、培养、获得、突出、相互作用、复杂/复杂性、关键、格局、关键性的、展示、织锦、证明、宝贵的、充满活力的 |
| **8. 系动词回避** | 用「作为/代表/标志着/充当」「拥有/设有/提供」替代简单「是/有」 |
| **9. 否定式排比** | 「不仅…而且…」「这不仅仅是…，而是…」过度使用 |
| **10. 三段式法则** | 强行将内容分成三组（主题演讲、小组讨论、社交机会；创新、灵感、行业洞察等） |
| **11. 同义词循环** | 同一意思用不同词重复（主人公→主要角色→中心人物→英雄） |
| **12. 虚假范围** | 「从 X 到 Y」但 X、Y 不在有意义的尺度上 |

**中文 AI 词汇分级（改写时优先处理）：**

| 层级 | 说明 | 示例 |
|------|------|------|
| **Tier 1（必改）** | 一出现即建议替换或删除 | 闭环、赋能、抓手、落地、痛点、打通、端到端、深度融合、有机结合、显著/有效提升、范式（空泛使用时）、底层逻辑（非专业语境） |
| **Tier 2（密集时改）** | 同段/同节出现 3 处以上时重点改 | 维度、场景、链路、协同、耦合、可复现、可扩展、系统性、整体性、全方位、一站式、颗粒度（非专业语境） |
| **Phrases（句式）** | 整句替换或拆解 | 「在…基础上/背景下/框架下」连用；「为…提供…/奠定…」；「首先…其次…最后…」「一是…二是…三是…」；「实现了…与…的有机结合/深度融合」 |

## 交流与填充

| 模式 | 需警惕的词汇/结构 |
|------|-------------------|
| **19. 协作交流痕迹** | 希望这对您有帮助、当然！、您说得完全正确！、请告诉我、这是一个… |
| **20. 知识截止免责** | 截至 [日期]、根据我最后的训练更新、虽然具体细节有限、基于可用信息… |
| **21. 谄媚语气** | 好问题！、您说得完全正确、这是一个很好的观点 |
| **22. 填充短语** | 为了实现这一目标→为了实现这一点；由于…的事实→因为…；在这个时间点→现在；值得注意的是数据显示→数据显示 |
| **23. 过度限定** | 可以潜在地可能被认为…可能会对…产生一些影响 → 简化为直接陈述 |
| **24. 通用积极结论** | 未来看起来光明、激动人心的时代、追求卓越的旅程、向正确方向迈出的一步 → 改为具体事实 |

## 快速检查清单（交付前自检）

- [ ] **连续三句长度相同？** 打断其中一句
- [ ] **段落总以简短句结尾？** 变换结尾方式
- [ ] **揭示前有破折号？** 删除
- [ ] **在解释隐喻/比喻？** 相信读者能理解，可删
- [ ] **「此外」「然而」等连接词堆砌？** 考虑删除或替换
- [ ] **三段式列举？** 改为两项或四项

## 处理流程（改写时）

**推荐处理顺序（多轮次，避免只改词不改结构）：**

1. **扫结构公式** — 每节段落数/列表项数是否雷同，是否每段都以「总—分—总」或相同句式结尾，是否重复「综上所述/值得注意的是/鉴于此」式收束；公式化「挑战与展望」改为具体事实。
2. **扫意义膨胀与宣传语** — 删除夸大意义、宣传/广告语、模糊归因（见上表模式 1–6）。
3. **扫 AI 词汇与句式** — 替换 Tier 1 必改词、密集出现时改 Tier 2；打破系动词回避、否定排比、三段式、同义循环、虚假范围（模式 7–12）。
4. **扫节奏与填充** — 句长变化、破折号/粗体/列表格式、交流痕迹与填充短语、过度限定、通用积极结论（模式 13–24）。
5. **用快速检查清单自检** 后再输出。

**执行步骤：**

1. 通读输入，识别上述模式实例  
2. 按 5 条核心规则重写问题片段  
3. 确保：句长有变化、用具体细节替代模糊主张、语气一致、适当用简单「是/有」  
4. **改写后说明**：若提供【修改说明】，至少有一条标明命中的模式（如「模式 7 + 模式 22」或「AIV + 填充短语」），便于审查与复检  
5. 用快速检查清单自检后再输出

## 质量评分（可选，总分 50）

| 维度 | 10 分 | 1 分 |
|------|-------|------|
| **直接性** | 直截了当 | 充满铺垫 |
| **节奏** | 长短交错 | 机械重复 |
| **信任度** | 简洁明了 | 过度解释 |
| **真实性** | 自然流畅 | 机械生硬 |
| **精炼度** | 无冗余 | 大量废话 |

- **45–50**：优秀，已去 AI 痕迹  
- **35–44**：良好，仍有改进空间  
- **&lt;35**：建议重新修订

## 个性与灵魂（避免「干净但无灵魂」）

- 句长与结构要有变化；有观点而非只中立报道；可承认不确定性或复杂感受；适当用第一人称；避免每段都是「总—分—总」。
- 学术论文中可在保持客观的前提下，用具体数据/方法/限定替代空泛表述，避免读起来像新闻稿或维基体。

**工学论文适配：**

- **有观点**：对方法/结果做判断（如「在所选场景下更优」「结论受限于算例规模」），而非仅罗列「A 方法如何、B 方法如何」。
- **一条限定语**：每条主张最多一处审慎限定（如「在多数情况下」「现有证据表明」），避免堆叠「可能」「或许」「在一定程度上」。
- **口语自检**：改写后可自问「这句话口头上会不会这么说？」若不会，改为更直接的说法。
