## 2. 人味化改写：推荐 Pass 顺序

按以下顺序扫稿，避免只改词不改结构：

| Pass | 做什么 | 对照 |
|------|--------|------|
| **1. 结构公式** | 每节段落数/列表项数是否雷同；是否每段「总—分—总」结尾；是否重复「综上所述/值得注意的是/鉴于此」式收束；公式化「挑战与展望」→ 改为具体事实 | humanizer-zh 模式 6、6.5 |
| **2. 意义膨胀与宣传语** | 删除夸大意义、宣传/广告语、模糊归因 | 模式 1–6 |
| **3. AI 词汇与句式** | Tier 1 必改；Tier 2 同段≥3 处则改；Phrases 句式拆解或替换 | 模式 7–12、中文词汇分级表 |
| **4. 节奏与填充** | 句长变化、破折号/粗体/列表格式、交流痕迹与填充短语、过度限定、通用积极结论 | 模式 13–24 |
| **5. 快速检查清单** | 用 humanizer-zh 文末清单自检后再输出 | humanizer-zh 快速检查清单 |

---

## 3. 人味化优先级（执行时牢记）

1. **先删后换**：先删填充短语与金句、破公式与节奏，再替换惯用词。
2. **有观点、落到具体**：对方法/结果做判断（如「在所选场景下更优」「结论受限于算例规模」）；用具体数据/方法/场景替代空泛表述。
3. **一条限定语**：每条主张最多一处审慎限定（如「在多数情况下」「现有证据表明」），不堆叠「可能」「或许」「在一定程度上」。
4. **口语自检**：改写后自问「这句话口头上会不会这么说？」若不会，改为更直接说法。
5. **大声朗读**：不顺即改。

## 4. AI 痕迹检测：层次与必检项

执行 Part C 或单独「AI 风格审查」时，按以下层次进行（可与 ai_detector.py / text_analyzer.py 结果结合）：

| 层次 | 内容 | 说明 |
|------|------|------|
| **L1 技术痕迹** | Markdown 误入非 Markdown 上下文；utm_source=chatgpt；oaicite/turn0search 等 | 若存在→确凿 AI 参与 |
| **L2 词汇** | 重要性/象征性短语、Tier 1/2 词、过用短语（「值得注意的是」「此外」「然而」等） | 见 §8 词表与 humanizer-zh |
| **L3 结构** | 虚假范围、同义循环、否定排比、模板结论、段落/句长过于均匀、破折号滥用、三项并列 | humanizer-zh 模式 9–12、13 |
| **L4 内容** | -ing 肤浅分析、宣传语、「挑战与未来展望」公式、重要性膨胀 | 模式 3、4、6 |
| **L5 连接词密度** | 总结/转折/结构连接词（综上所述、此外、然而、首先其次最后等）每 3–4 句一个→疑 | 中文/朱雀向 |
| **L6 格式** | 内联标题列表、过度粗体、Title Case 标题、表情符号 | 模式 14–17 |
| **L7 语气** | 过度礼貌与 hedging、风格无变化、缺第一人称、全篇被动 | 模式 19–24 |
| **结论** | 需 3+ 信号再下「疑似 AI」；列出缓解因素（具体数据、领域术语、个人判断等） | 见 §7 报告片段 |

**误判排除**：公式与术语定义、引用句、代码/命令不单独因「工整」判为 AI 味。

## 5. 改写后输出要求

- 润色/改写结果保存到 `my-thesis/`，命名 `第X章_主题_年月日.md`。
- **【修改说明】**：至少一条标明命中的 humanizer-zh 模式或 Part C 类型（如「模式 7 + 模式 22」「AIV + 填充短语」），便于复检。
- 可选：质量评分 5 维度 /50 分（直接性、节奏、信任度、真实性、精炼度），45+ 为优，&lt;35 建议再改。

---

## 6. 改写完成检查清单（交付前）

- [ ] 按 Pass 1→5 顺序扫过一遍
- [ ] 句长有变化（短/中/长结合）
- [ ] 机械过渡词已减少或替换（首先/其次/此外/然而/综上所述等）
- [ ] 抽象套话已替换为具体概念或术语
- [ ] 至少一处具体方法/参数/场景或学术限定
- [ ] 无连续三句同长、破折号不过多、无三段式堆砌、连接词未堆砌
- [ ] 段落结尾方式有变化（不每段简短句收束）
- [ ] 修改说明中至少一条标注模式/类型
- [ ] 原意与引用、数据完整保留；学科用语符合工学/运筹习惯

## 7. 检测报告与问题标注

**Part C 问题类型**（与 humanizer-zh 模式对应）：

| 类型 | 含义 | 对应模式示例 |
|------|------|----------------|
| AIT | AI 典型过渡 | 机械过渡词、模式 7 连接词 |
| AIS | AI 典型句式 | 抽象套话、模式 22–24 |
| AIV | AI 惯用词 | Tier 1/2、模式 7 |
| AIF | 惯用句式/结构 | 模式 9–12、6.5 |
| AIW | 词汇单一 | 同段重复用词、模式 11 |
| AIP | 结构雷同 | 模式 6.5、10、段落同构 |

**报告片段（可并入 Part C 产出）**：

- **证据**：按层次列出（例：L2 词汇：出现「此外」「格局」；L3 结构：连续三句句长相近）。
- **缓解因素**：具体数据、算法名、参数、限定语、领域术语等。
- **建议**：具体改写方向（如「改为具体结论/数据」「缩短句长」「替换过渡词」）。

**统计指标提示**（若运行了 ai_detector / text_analyzer）：在报告中可加一句「句长分布/过渡词密度/TTR：…」，与人工审查结论并列。

## 8. 速查表

### 中文 AI 词汇分级

| 层级 | 处理 | 示例 |
|------|------|------|
| **Tier 1（必改）** | 一出现即改 | 闭环、赋能、抓手、落地、痛点、打通、端到端、深度融合、有机结合、显著/有效提升、范式（空泛时）、底层逻辑（非专业时） |
| **Tier 2（密集时改）** | 同段/同节≥3 处则改 | 维度、场景、链路、协同、耦合、可复现、可扩展、系统性、整体性、全方位、一站式、颗粒度（非专业时） |
| **Phrases（句式）** | 整句替换或拆解 | 「在…基础上/背景下/框架下」连用；「为…提供…/奠定…」；「首先…其次…最后…」「一是…二是…三是…」；「实现了…与…的有机结合/深度融合」 |

### 必杀句式（出现即标或改）

- **对比公式**：「不是…而是…」「这不仅仅是…，而是…」「The problem isn't X—it's Y」
- **开场拐杖**：「值得注意的是」「在…的今天」「从…来看」「Here's the thing:」「Let me be clear:」
- **强调拐杖**：「That's it. That's the [noun].」「Enter: [thing]」「And here's the kicker」

### 人味信号（加分项，可保留或增强）

- 对方法/结果有明确判断；具体数据、算法名、参数；适当限定（每条主张一处）；句长与段落长度有变化；视角自然（我/我们/被动按需切换）。
