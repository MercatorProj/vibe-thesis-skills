# 工作流程指南（格式化/润色模式）

## 技能使用流程

### 步骤1: 接收用户输入

当用户提供以下类型的内容时,触发本技能:
- 草稿段落文本
- 技术要点列表
- 参考文献内容
- 需要润色的学术文本

### 步骤2: 分析输入内容

识别以下关键信息:
- **章节归属**: 判断内容属于第3-6章的哪一章
- **内容类型**: 通行性分析/路径规划/调度优化/实验验证
- **技术要素**: 是否包含数学公式、算法描述、实验数据等
- **润色需求**: 语言优化/公式规范化/术语统一/逻辑重构

### 步骤3: 执行润色处理

#### 3.1 语言风格学术化

**转换规则**:
- 口语化表达 → 正式学术表达
- 简单句 → 复杂句(保持清晰)
- 重复表述 → 精简优化
- 模糊表达 → 精确表达

**示例**:
- "这个方法很好用" → "该方法具有良好的适用性和有效性"
- "我们做了实验" → "本文开展了系统的实验验证"
- "结果不错" → "实验结果表明该方法具有显著优势"

#### 3.2 术语规范化

**操作流程**:
1. 识别文中的专业术语
2. 查阅 `references/terminology.md` 确认标准表达
3. 统一全文术语使用
4. 确保术语一致性

**注意事项**:
- 首次出现术语时给出全称和缩写
- 后续使用统一使用缩写
- 避免同一概念使用不同表述

#### 3.3 数学公式规范化

**处理步骤**:
1. 识别文中的数学公式
2. 使用 `references/formula-templates.md` 中的标准格式
3. 补充公式符号说明

**公式处理清单**:
- [ ] 使用 `$$ ... $$` 包裹独立公式
- [ ] 使用 `$ ... $` 包裹行内公式
- [ ] 补充完整的符号说明
- [ ] 规范化单位表示
- [ ] 检查LaTeX语法正确性

#### 3.4 逻辑优化

**优化策略**:
- 段落开头: 明确主题句
- 段落中间: 展开论述,提供论据
- 段落结尾: 总结或过渡

**逻辑连接词使用**:
- 递进: "首先...其次...最终"
- 转折: "然而...不过..."
- 因果: "因此...基于此..."
- 补充: "此外...同时..."

### 步骤4: 输出Markdown文件

#### 4.1 文件命名规则

格式: `第X章_主题_年月日.md`

示例:
- `第3章_通行区识别_20260206.md`
- `第4章_A星算法设计_20260206.md`
- `第5章_VRP模型构建_20260206.md`
- `第6章_实验结果分析_20260206.md`

#### 4.2 文件保存位置

默认保存路径: `工作目录/my-thesis/`

#### 4.3 文件内容要求

- **纯段落格式**: 不包含任何Markdown标题(#, ##等)
- **连续文本**: 生成可直接插入论文的完整段落
- **格式正确**: 确保Markdown语法无误
- **公式规范**: 使用标准LaTeX格式

### 步骤5: 质量检查

#### 检查清单

**学术规范性**:
- [ ] 符合学位论文写作标准
- [ ] 使用正式学术语言
- [ ] 术语使用统一规范
- [ ] 逻辑连贯清晰

**技术正确性**:
- [ ] 技术描述准确无误
- [ ] 数学公式正确
- [ ] 单位表示规范
- [ ] 符号说明完整

**格式完整性**:
- [ ] Markdown语法正确
- [ ] 无格式错误
- [ ] 文件命名规范

## 参考资源使用

### 何时查阅参考文件

**formatting-examples.md**:
- 需要参考润色示例时
- 不确定如何表达某些技术内容时

**terminology.md**:
- 需要确认术语标准表达时
- 发现术语使用不一致时

**formula-templates.md**:
- 需要规范化数学公式时
- 不确定LaTeX语法时
- 需要查找特定公式模板时

## 必须遵守的原则

1. **每次都输出MD文件**: 不论用户是否明确要求,必须输出Markdown文件
2. **纯段落格式**: 不包含任何标题层级
3. **学术风格**: 始终保持正式的学术语言
4. **公式规范**: 使用标准LaTeX格式和编号
5. **文件命名**: 使用中文命名,格式为 `第X章_主题_年月日.md`

### 常见错误避免

- ❌ 在输出中添加Markdown标题
- ❌ 使用口语化表达
- ❌ 忘记补充符号说明
- ❌ 术语使用不一致
- ❌ 不输出MD文件
