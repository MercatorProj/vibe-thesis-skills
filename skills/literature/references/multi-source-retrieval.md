# 多源检索与题录合并（可重复检索）

本说明配合 `literature` 技能使用：在依赖 Zotero 或模型泛搜之外，通过**各数据库导出 → 本地合并 → 去重**形成可审计的题录池，便于复现检索策略与纳入文献范围。

## 适用场景

- 学位论文/期刊综述需要写明「检索库、时间窗、检索式」；
- 单一来源（仅 Zotero 或仅某网页）不足以覆盖主题；
- 需要与协作者共享**同一批题录**（RIS/Bib），而非口头列表。

## 推荐流程

1. **固定检索式**：中英文关键词、布尔算符（AND/OR/NOT）、字段限制（题名/摘要等）；记录**检索日期**与**数据库名称**。
2. **分库导出**：在各库使用相同或等价的检索式（注意不同库语法差异），导出 **RIS** 或 **BibTeX**（优先带 DOI、作者、年份、题名）。
3. **导入文献管理器（可选）**：将导出文件导入 Zotero/EndNote 等，便于去重与全文管理；亦可跳过，直接合并纯文本题录。
4. **合并与去重**：将多份 `.ris`/`.bib` 合并为一份主表；按 **DOI**（优先）或 **规范化标题 + 年份** 去重。可使用本技能目录下 `scripts/dedupe_bibtex.py`（仅 BibTeX 合并场景）。
5. **进入写作链路**：对合并后的条目用 `literature` 的「阅读与综述」「生成 BibTeX」等模式继续。

## 常见数据库导出要点（操作级，不含账号）

以下仅为**一般性操作提示**，各平台界面会变；以各库官方帮助为准。

| 来源 | 导出思路 |
|------|----------|
| **Web of Science / Scopus** | 检索结果页选择记录 → Export / 导出 → RIS 或 BibTeX；注意记录数上限与去重说明。 |
| **Google Scholar** | 单条可点「引用」→ BibTeX；批量需借助 Scholar 设置或第三方工具，**注意使用条款**。 |
| **CNKI（中国知网）** | 检索结果勾选 → 导出/参考文献 → 选择 Refworks、EndNote、NoteExpress 等格式（常为 RIS 兼容）；注意机构权限。 |
| **IEEE Xplore / ACM / Springer** | 检索结果多选 → Download citations → BibTeX 或 RIS。 |
| **PubMed** | Send to → Citation manager → 创建文件（Medline 格式可再转）。 |

## 去重规则建议

1. **有 DOI**：同一 DOI 只保留一条；冲突时保留字段更完整的一条。
2. **无 DOI**：用「标题（小写、去标点合并空格）+ 首作者姓 + 年份」作为弱键；人工复核疑似重复。
3. **预印本与正式刊**：同一工作可能两条记录，需在综述中显式说明保留策略。

## 与 Zotero 脚本的关系

- `zotero_searcher.py`：针对**已在本机 Zotero 库中**的条目检索；
- 本节：针对**从各库新导出、尚未入库**的批量题录。二者可先后使用：导出合并去重 → 导入 Zotero → 再用脚本导出 `.bib` 供 LaTeX。

## 与系统综述筛选

若项目为**系统综述**并需纳入/排除记录与 PRISMA 流程说明，在题录合并之外请使用 **`systematic-review-lite`** 技能中的模板与检查项。
